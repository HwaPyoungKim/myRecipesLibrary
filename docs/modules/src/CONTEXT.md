# CONTEXT

## Qué hace

`src` contiene toda la aplicación: rutas, componentes, datos, assets, estilos, hooks, utilidades y árbol de rutas generado.

## Qué problema resuelve

Separa la experiencia de usuario de la configuración raíz. Todo lo que afecta al producto visible y al runtime React vive aquí.

## Cómo usarlo

- Crear páginas en `src/routes`.
- Reutilizar piezas visuales desde `src/components`.
- Mantener datos de recetas en `src/data`.
- Importar imágenes desde `src/assets`.
- Usar `@/` como alias hacia `src`.

## Mapa mental

`routes` decide qué se muestra, `data` provee contenido, `components` lo presenta, `assets` entrega imágenes y `styles.css` define el lenguaje visual.
